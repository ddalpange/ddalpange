    - 프론트엔드-백엔드 로직 통일로 스키마 불일치 고통 해소
- 진단사 웹뷰 분리
    - 독립 프로젝트로 분리하여 컴파일 시간 단축
    - 배포 안정성 및 독립성 확보
- 릴레이 Fragment 기반 진단리포트 중앙 관리
    - 멀티프로젝트에서 공통 데이터 패칭 로직 중앙집중화
    - 프로젝트별 데이터 위계 불일치 문제 해소 → 클레임 리스크 제거
